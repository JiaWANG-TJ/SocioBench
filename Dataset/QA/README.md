# 问卷数据格式化工具

## 功能说明

这个工具用于将Excel格式的问卷数据转换为标准的JSON数据集。转换后的JSON数据具有以下格式：

```json
{
  "code": "V2014001",  // V+问卷年（四位数）+ 编号（三位数）
  "topic": "Citizenship",  // 问题领域
  "question": "问题文本",  // 问题详细描述
  "option_code": ["1", "2", "3", "4", "5", "6", "7", "8"],  // 选项代码
  "option_text": [  // 选项文本
    "Not at all Important",
    "",
    "",
    "",
    "",
    "",
    "Very Important",
    "Can't Choose"
  ]
}
```

## 数据集信息

当前支持的主题及其年份：
- Citizenship（公民）- 2014年
- Environment（环境）- 2020年
- Family（家庭）- 2012年
- Health（健康）- 2021年

问题编码格式为"V年份三位数编号"，例如：
- Citizenship: V2014001, V2014002, ...
- Environment: V2020001, V2020002, ...
- Family: V2012001, V2012002, ...
- Health: V2021001, V2021002, ...

## 使用方法

1. 将Excel问卷数据文件放置在此目录下
2. 执行脚本进行转换

```bash
python excel_to_json.py
```

3. 脚本将自动处理目录下的Excel文件，并生成对应的JSON文件

## 输入文件要求

Excel文件应遵循以下结构：
- 列名行：第一列为问题，第二列为选项
- 数据行：每行包含一个问题及其对应的选项

## 支持的选项格式

脚本能够处理以下几种选项格式：

1. 代码在前，文本在括号中，如：
   - `1 (Not at all Important)`
   - `7 (Very Important)`
   - `8 (Can't Choose)`

2. 文本在前，代码在括号中，如：
   - `Health care (1)`
   - `Education (2)`
   - `The environment (4)`

3. 特殊格式：代码和文本均在括号中，如：
   - `(-8) (Can't choose)`

这些格式都会被正确解析为选项代码和选项文本。负数代码（如-8）也能正确处理。

## 输出说明

脚本将为每个输入的Excel文件生成一个对应的JSON文件，命名格式为`{主题}_qa.json`。

## 注意事项

1. 脚本会自动处理Excel列名作为第一个问题
2. 脚本为问题生成编码，格式为"V年份三位数编号"，编号从001开始
3. 选项代码和选项文本会被分别提取并存储在不同的数组中
4. 无法解析的选项格式会将整个文本作为选项文本，代码为空字符串
